2 Main Agent — ei_graph (9 Nodes)

Every user message passes through all 9 nodes in sequence. The safety node is the only branching point — critical messages skip coaching and go straight to persist. The evaluate → refine loop auto-improves low-quality responses.

Personalisation
load_profile
Reads Firebase, returns UserProfile with name, triggers, preferred techniques.
Memory
load_memory
Fetches last 5 sessions, uses GPT-4o-mini to compress into 2-3 sentences of context.
Analysis
analyze
Emotion extraction (GPT) + toxicity score (Perspective API) run in the same node.
Safety Gate
safety
Only branching point. Critical → skip coaching, show crisis resources + hotlines.
Coaching
coach
Richest prompt in the graph. Receives profile + memory + emotion + toxicity for full context.
Quality Gate
evaluate → refine
Scores empathy/specificity/safety. One auto-refinement pass if any score is below threshold.
Reflection
journal
Generates a single open-ended journaling question personalised to the current emotion.
Persistence
persist
Writes session entry to Firestore. Updates user profile stats and emotion summary.

3 Pattern Agent — pattern_graph (On Demand)

Triggered manually via the sidebar "Analyse My Patterns" button or CLI --patterns flag. Loads up to 30 sessions and uses GPT-4o-mini to surface recurring patterns, triggers, trend direction, and a long-term recommendation.

4 Voice Emotion Pipeline

Voice input is pre-processed before entering the main graph. Transcription is always attempted (no key needed). Hume prosody is additive — coaching always uses the transcribed text; prosody scores are shown alongside it in the UI.

5 EIState — Data Flow Through the Graph

The LangGraph state (EIState) is a TypedDict that accumulates data as each node runs. Each node reads what it needs and writes back its output — the final state contains everything.

Field Type Set by Node Used by Node(s) Description
user_id str Input (app.py) load_profile · load_memory · persist Stable identifier derived from user's name (e.g. "john_doe")
user_name str Input (app.py) load_profile · coach Display name — used when creating a new profile
user_text str Input (app.py) analyze · safety · coach · evaluate · refine · journal The user's raw message (or voice transcription)
session_id str Input (app.py) persist UUID generated per browser session — groups entries in Firestore
user_profile dict load_profile coach · persist UserProfile: name, known_triggers, preferred_techniques, emotion_summary
memory_context str load_memory coach · journal 2-3 sentence GPT summary of last 5 sessions' emotions and patterns
emotion dict analyze safety · coach · evaluate · refine · journal · persist EmotionMetrics: valence(-1→1), arousal(0→1), primary_emotions[], confidence, rationale
toxicity float|None analyze coach · evaluate · persist Google Perspective API score 0–1. None if API unavailable.
safety dict safety persist SafetyLabel: is_critical, category, confidence, reason, crisis_message
is_critical bool safety Router (conditional edge) True → skip coaching, show crisis message, go straight to persist
coaching str coach / refine evaluate · refine · persist · UI 3-line personalised coaching response (may be replaced by refine node)
eval dict evaluate refine · persist · UI EvalScores: empathy, specificity, safety (each 0–1), revise (bool), critique (str)
journal_prompt dict|None journal persist · UI JournalPrompt: prompt (question), emotion_context, suggested_duration_minutes

6 Skills & Services Reference

Every skill is a pure Python function — no class, no state. They receive typed inputs and return typed outputs. All LLM calls go through backend/services/anthropic_client.py (OpenAI singleton).
